SQLite Agentic RAG Sample Design

Recommended Dataset

Use Student Management because it naturally mixes structured analytics and unstructured evidence.

  • Structured SQLite tables: students, courses, enrollments, attendance, assessments, fees, advising_cases.
  • Unstructured documents for embeddings: advising notes, course descriptions, scholarship policy, attendance policy, intervention playbooks.
  • Example questions:
    • Which students are at risk this term and why?
    • Show average grade by course and explain weak areas.
    • Who qualifies for scholarship support based on GPA, attendance, and fee status?
    • Create a chart of attendance trend by month for at-risk students.

Alternative datasets: retail orders, clinic appointments, or HR training.

Target Architecture

Implementation Plan

  1. Add a small sample dataset under data/student_management/ with CSV seed files and text or Markdown notes and policies for embedding retrieval.
  2. Add scripts/build_student_db.py to create student_management.sqlite from the seed files using Python sqlite3, including useful views such as student_risk_summary.
  3. Add vector ingestion for unstructured student data using HuggingFace embeddings and a separate Chroma directory such as chroma_student_db/.
  4. Add src/student_rag/ as the source package with focused modules for agents, SQLite, retrieval, artifacts, LM Studio configuration, and paths.
  5. Add student_rag.agent with tools for plan_question, run_sql, retrieve_notes, generate_table_or_chart_spec, replan, and answer.
  6. Add student_rag.mcp_server so LM Studio Chat can call local Student Management tools through MCP.
  7. Add demo questions and evaluation examples in eval/student_questions.json.
  8. Update docs with setup, workflow diagram, dataset description, and example prompts.

Scope Boundary

Keep the first version local and lightweight: SQLite from the standard library, Chroma for embeddings, LM Studio through the existing OpenAI-compatible config, and chart output as a simple Vega-Lite JSON spec or saved PNG only if visualization is needed in the demo.

Current MCP Tools

  • ask_student_management: default plain-language tool that routes common questions to the right SQL and vector evidence internally.
  • get_schema_summary: returns tables, views, columns, and value hints.
  • run_sql: executes one read-only SQL query and returns warnings for suspicious values such as scholarship_candidate = 'yes'.
  • get_at_risk_students: returns high-risk and medium-risk students using the correct risk rules.
  • analyze_at_risk_students: returns at-risk student rows plus advising notes and intervention policy context.
  • get_scholarship_candidates: returns scholarship candidates using scholarship_candidate = 1.
  • analyze_scholarship_candidates: returns scholarship candidate rows plus scholarship policy context.
  • retrieve_notes: searches advising notes, policies, and course descriptions.
  • generate_artifact: creates a Markdown table or Vega-Lite chart spec from SQL results.

Schema Value Hints

  • student_risk_summary.risk_level uses high, medium, and low.
  • student_risk_summary.scholarship_candidate is an integer flag: 1 means yes and 0 means no.
  • student_risk_summary.term uses values such as 2026-Spring; there is no current_term.
  • fee_summary.status uses paid, partial, and overdue.
