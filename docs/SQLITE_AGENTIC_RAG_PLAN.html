SQLite Agentic RAG Sample

Recommended Dataset

Use Student Management because it naturally mixes structured analytics and unstructured evidence.

  • Structured SQLite tables: students, courses, enrollments, attendance, assessments, fees, advising_cases.
  • Unstructured documents for embeddings: advising notes, course descriptions, scholarship policy, attendance policy, intervention playbooks.
  • Example questions:
    • Which students are at risk this term and why?
    • Show average grade by course and explain weak areas.
    • Who qualifies for scholarship support based on GPA, attendance, and fee status?
    • Create a chart of attendance trend by month for at-risk students.

Alternative datasets: retail orders, clinic appointments, or HR training.

Target Architecture

Implementation Plan

  1. Add a small sample dataset under data/student_management/ with CSV seed files and text or Markdown notes and policies for embedding retrieval.
  2. Add scripts/build_student_db.py to create student_management.sqlite from the seed files using Python sqlite3, including useful views such as student_risk_summary.
  3. Add vector ingestion for unstructured student data using HuggingFace embeddings and a separate Chroma directory such as chroma_student_db/.
  4. Add student_rag.agent with tools for plan_question, run_sql, retrieve_notes, generate_table_or_chart_spec, replan, and answer.
  5. Add demo questions and evaluation examples in eval/student_questions.json.
  6. Update docs with setup, workflow diagram, dataset description, and example prompts.

Scope Boundary

Keep the first version local and lightweight: SQLite from the standard library, Chroma for embeddings, LM Studio through the existing OpenAI-compatible config, and chart output as a simple Vega-Lite JSON spec or saved PNG only if visualization is needed in the demo.
