Student Management Data Design

This document describes the data layer for the Student Management Agentic RAG sample. All seed content lives under data/student_management/ as CSV files. Build scripts materialize SQLite for structured analytics and Neo4j for policy and intervention relationships.

Source of truth: edit CSV files, then rebuild. python scripts/build_student_db.py and python scripts/build_student_kg.py. Do not hardcode policy thresholds in Python source.

1. High-Level Data Flow

CSV seed files SQLite runtime store Neo4j knowledge graph

Operational and policy CSV files load into SQLite. Policy and advising link CSV files also feed Neo4j through extraction.py. policy_loader.py reads policy_rules.csv and generates the student_risk_summary view SQL at database build time.

6.1 Responsibility split

7. Policy Change Procedure

  1. Edit the relevant CSV under data/student_management/.
  2. Rebuild SQLite: python scripts/build_student_db.py
  3. Rebuild Neo4j: python scripts/build_student_kg.py
  4. Re-run agents or MCP tools against the refreshed stores.

7.1 Common change examples

ChangeEdit fileRebuild
Attendance high-risk threshold 75 → 70 policy_rules.csv row R002 SQLite only
New intervention playbook item interventions.csv, policy_intervention_links.csv SQLite + Neo4j
Advisor note for a student advising_notes.csv, optional student_risk_factors.csv SQLite + Neo4j if graph links changed
New course description courses.csv SQLite; Neo4j only if graph links reference the course
